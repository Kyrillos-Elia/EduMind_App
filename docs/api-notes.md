# 🤖 AI API Notes – AI Study App (Detailed)

## 📌 Purpose

This file documents the AI-related integrations in the project: how the app calls AI-driven services (currently using `firebase_ai` generative models), how PDFs are handled, expected request/response shapes, security considerations, and recommendations for production.

---

## 🔗 Summary of Current Implementation

- `lib/services/pdf_ai_service.dart` implements the PDF → AI pipeline.
  - Uses `file_picker` to select PDFs on device/web.
  - Uses `syncfusion_flutter_pdf` to extract text from PDF bytes.
  - Uses `firebase_ai` (GenerativeModel) to call an on-platform generative model `gemini-2.5-flash`.
  - Provides `generateSummary(String pdfText)` and `generateQuiz(String pdfText)` helpers.
- `lib/screens/al-chat.dart` is a chat UI with placeholder responses (no network calls yet).

This file documents how these pieces work and how to extend them.

---

## 🧭 Pipeline Overview (PDF → AI)

1. User picks a PDF via `FilePicker` (`pickAndUploadPdf()` in `PdfAiService`).
2. The app reads `Uint8List` bytes and extracts text using `PdfTextExtractor` (Syncfusion).
3. The extracted text is sent to the generative model via `FirebaseAI.generativeModel(...).generateContent(...)`.
4. The response text is parsed and returned to the UI.

Notes:

- Large PDFs must be chunked before calling the model (models have context/token limits).
- The current implementation sends the full extracted text in prompts — this works for small documents but is unsafe for big files.

- The `generateQuiz` implementation strips common code fence markers and decodes JSON. It expects the model to return strict JSON.

## ✅ Dart usage examples (how to call from UI)

- Summary:

```dart
final pdfService = PdfAiService();
final picked = await pdfService.pickAndUploadPdf();
if (picked != null) {
  final text = picked['text'] as String;
  final summary = await pdfService.generateSummary(text);
  // show `summary` in UI
}
```

- Generate quiz:

```dart
final questions = await pdfService.generateQuiz(pdfText);
// `questions` is List<QuizQuestion>
```

## 🔧 Recommendations & Production Hardening

1. Chunking & Summarization Strategy

- For large PDFs: split text into smaller chunks (e.g., 2–4 KB of text) and either:
  - Summarize each chunk and then summarize the summaries (hierarchical summarization), or
  - Generate embeddings for each chunk and retrieve top-k relevant chunks for question-specific prompts.

2. Robust JSON Handling

- Do not rely on raw `response.text` to be exact JSON. Wrap parsing with try/catch and use heuristics to extract JSON blocks (strip markdown fences, trim non-json prefix/suffix).
- Add validation for the parsed JSON shape and fallback gracefully if malformed.

3. Rate-limiting & Retries

- Implement client-side throttling to avoid excessive model calls (debounce chat typing, disable repeated resend buttons).
- Use exponential backoff and a max attempt limit for transient errors.

4. Cost & Token Control

- Use shorter prompts where possible.
- Limit chunk sizes to control token usage.
- Consider using cheaper models for summarization vs. high-quality models for complex reasoning.

5. Security

- Never hardcode API keys in the app.
- Prefer one of:
  - Use `firebase_ai` (server-managed) with proper platform config, or
  - Proxy requests through a secure backend or Firebase Cloud Function that stores API keys and enforces quotas and authentication.
- Ensure the Firebase project has billing enabled and IAM policies locked down for production models.

6. Privacy & PII

- Avoid sending user PII to third-party APIs unless required and disclosed.
- If storing user content, use Firebase Firestore/Storage with appropriate security rules and retention policies.

## ☁️ Configuration & Environment

- The repo includes `firebase_options.dart` (generated by FlutterFire). Ensure it matches your Firebase project.
- If switching to another AI provider (OpenAI, Anthropic, etc.), create environment variables or server-side secrets and do NOT commit them.

Example (server-side env vars):

```
OPENAI_API_KEY=sk-...
AI_SERVICE_URL=https://api.openai.com/v1/chat/completions
```

If using Firebase Functions as a proxy, the function should validate incoming requests (ensure user auth via Firebase ID token), then call the AI provider.

---

## 🧪 Error Handling & Edge Cases

- Detect empty or very short extracted text and show a friendly message: "Could not extract text from PDF or file is empty."
- For JSON parsing errors from `generateQuiz`, fall back to:
  - Attempt to extract JSON substring using regex (first `[` ... last `]`), then parse.
  - If parsing still fails, show an error message and offer a manual question creation UI.

## 🔁 Chat Integration (future)

- `lib/screens/al-chat.dart` currently uses placeholder responses.
- To integrate a real chat model:
  - Add a chat service wrapper (similar to `PdfAiService`) that sends messages and receives streaming or batched responses.
  - Consider streaming responses for better UX (partial replies while model is generating).

Chat message shape (example for serverless proxy):

Request:

```json
POST /v1/chat
Headers: Authorization: Bearer <token>
Body: { "messages": [{"role":"user","content":"Explain binary trees"}] }
```

Response (example):

```json
{
  "id": "chatcmpl-...",
  "choices": [
    { "message": { "role": "assistant", "content": "Binary trees are..." } }
  ]
}
```

Client side should map messages to the `Message` model used in `al-chat.dart`.
