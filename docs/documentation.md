# 📘 AI Study App – Complete Documentation (Updated)

## 📌 Overview

`AI Study App` is a Flutter mobile application that provides students with an AI-powered study companion. It combines a polished UI (Figma-based), Firebase authentication, PDF summarization UI, an AI chat placeholder, and user profile & tracking features. This document summarizes the current codebase, runtime flow, routes, dependencies, and recent updates so a reviewer (e.g., instructor) can understand the project at a glance.

## 🔗 Routing & App Flow

- `main.dart` initializes Firebase and uses `AuthWrapper` to determine the initial route:
  - If no user → `LoginScreen`
  - If user exists and `emailVerified` → `MainNavigation` (tabs)
  - If user exists and NOT verified → `EmailVerificationScreen`

- Routes registered in `MaterialApp`:
  - `/onboarding` → `OnboardingScreen`
  - `/login` → `LoginScreen`
  - `/home` → `MainNavigation`
  - `/verify-email` → `EmailVerificationScreen`

## ⚙️ Email verification flow (details)

- When a logged-in user is not verified, `AuthWrapper` routes to `EmailVerificationScreen`.
- `EmailVerificationScreen` behavior:
  - Sends a verification email automatically on first open.
  - Starts a periodic timer (3s) to reload the Firebase user and check `emailVerified`.
  - If verified → navigates to `/home` (MainNavigation).
  - `Resend Email` button triggers `sendEmailVerification()` with a short cooldown (5s) and shows a top toast for 3s.

Files of interest: `lib/screens/EmailVerificationScreen.dart` (redesigned with animated background, toast, and glow icon).

## 🧭 Navigation & Keeping Bottom Bar Visible

- The bottom navigation is implemented in `MainNavigation` using an `IndexedStack`.
- `HomeScreen` was updated to accept an optional `onProfileTap` callback; when provided the top-right avatar triggers the callback instead of pushing a new `ProfilePage`, allowing the app to switch to the Profile tab (`_currentIndex = 3`) while keeping the bottom navigation visible.

Files changed recently (see "Recent changes" below) relate to this behavior.

## 🛠️ How to Run Locally

1. Ensure Flutter is installed (stable channel) and `flutter doctor` is clean.
2. Copy or ensure the correct Firebase config (the project already has `firebase_options.dart` — ensure you created the Firebase project and enabled Authentication).
3. Install dependencies:

```bash
flutter pub get
```

4. Run on device/emulator:

```bash
flutter run
```

Notes:

- If Firebase options were generated for a different bundle id/package name, regenerate using `flutterfire configure` or update `firebase_options.dart`.
- Android/iOS-specific setup (SHA keys, Google services files) must be present for Firebase Auth to work.
