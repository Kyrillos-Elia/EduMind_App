# 📚 AI Study App – Flutter Mobile Application

An intelligent, AI-powered educational platform built with **Flutter** that enables students to learn smarter. The app combines a polished, modern UI with Firebase authentication, interactive features (chat, PDF summarization, quiz generation), and user progress tracking.

**Status:** In active development | **Platform:** Android, iOS, Web

---

## 🎯 Project Overview

**EduMind** is a comprehensive study companion app that:

- ✅ Provides secure, email-verified user authentication
- ✅ Delivers an intuitive home hub with animated UI elements
- ✅ Offers AI-driven features: chat assistant, PDF summarization, quiz generation
- ✅ Tracks user progress and study streaks
- ✅ Supports both English and Arabic interfaces
- 🚧 Extends Firebase integration for real-time data sync

For detailed technical documentation, see [docs/documentation.md](docs/documentation.md) and [docs/api-notes.md](docs/api-notes.md).

## 🚀 Core Features (Implemented & Planned)

### ✅ Completed

- **Authentication:** Email/password signup & login with Firebase Auth
- **Email Verification:** Resend verification emails, automated polling, toast notifications
- **Navigation:** Bottom tab bar with seamless tab switching (Home, Tasks, Stats, Profile)

- **PDF Summarization:** Real backend integration for AI-powered summaries
- **Quiz Generation:** Auto-generate questions from uploaded PDFs
- **Firestore Integration:** Sync user data, saved summaries, quiz history
- **Cloud Storage:** Store user PDFs and generated content

## 🔗 App Flow & Routing

### Authentication Flow

1. **App Start** → `AuthWrapper` checks Firebase user status
   - No user → `LoginScreen`
   - User exists + email verified → `MainNavigation` (dashboard)
   - User exists + NOT verified → `EmailVerificationScreen`

2. **Routes** (registered in `main.dart`)
   ```dart
   '/onboarding'    → OnboardingScreen
   '/login'         → LoginScreen
   '/home'          → MainNavigation
   '/verify-email'  → EmailVerificationScreen
   ```

### Main Navigation Tabs

- **Home** – hub with orb, streak, quick actions
- **Tasks** – task management
- **Stats** – user progress & charts
- **Profile** – user info, settings, language selection

From any tab, tapping the avatar (top-right on Home) navigates to Profile tab while **keeping the bottom bar visible**.

## ⚙️ Setup & Installation

### Prerequisites

- Flutter SDK (stable channel, version 3.x+)
- Dart 3.x
- A Firebase project with Email/Password authentication enabled
- (Optional) A generative AI provider API key (OpenAI, Google AI, etc.)

### Step-by-Step Setup

1. **Clone the repository:**

   ```bash
   git clone https://github.com/demiana25/EduMind-Flutter-project.git
   cd EduMind-Flutter-project
   ```

2. **Check Flutter installation:**

   ```bash
   flutter doctor
   ```

   Ensure all checks pass (or at least, SDK, Android/iOS toolchain, and IDE are OK).

3. **Install dependencies:**

   ```bash
   flutter pub get
   ```

4. **Configure Firebase (if not already done):**

   ```bash
   flutter pub add firebase_core firebase_auth
   flutterfire configure
   ```

   This updates `firebase_options.dart` with your Firebase project settings. When prompted:
   - Select your Firebase project
   - Select platforms (Android, iOS, Web, etc.)
   - Accept the auto-generated configuration

5. **Build & Run:**

   ```bash
   # On Android emulator
   flutter run -d emulator-5554

   # On iOS simulator
   flutter run -d iphone

   # On physical device
   flutter run
   ```

6. **(Optional) Clean rebuild:**
   ```bash
   flutter clean
   flutter pub get
   flutter run
   ```

## 🔧 Configuration & Customization

### Theme & Appearance

- Dark mode is the default; toggle via `themeModeNotifier` in `theme_controller.dart`.
- Colors are centralized in `app_palette.dart` for easy rebranding.
- All sizes and spacing follow Material Design 3 standards.

### Firebase Setup

- Ensure Authentication (Email/Password) is enabled in your Firebase Console.
- (For AI features) Enable Firestore and Cloud Storage if storing user data.
- Add service account keys to `firebase_options.dart` or use FlutterFire CLI.

### Environment Variables (Future)

- For production AI APIs, use Firebase Cloud Functions or a backend proxy to store API keys securely.
- Never commit `.env` files or API keys to version control.
